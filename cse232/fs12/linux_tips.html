Linux Tips

Shell Tips

  1. End a text program that is hanging (i.e. in an infinite loop)
    Press ctrl-c (with the correct terminal in the foreground). This can often, but not always be used to escape from interactive programs as well.
  2. End a misbehaving graphical program
    Use “ps” to find its “process id” (also printed in terminal when programs started), then “kill [process id]”. Despite the name, this asks the program nicely to end. If it doesn't work, try "kill -9 [process id]" which forces it to end.
  3. Run a graphical programs from the command line
    Add an '&' at the end of the command to fork the process (run the program in a new process). I you do this, you can press enter to get a prompt and continue using the terminal. If you don't, the prompt won't return until you close the program or enter ctrl-c, and closing the terminal also kills the graphical program.
  4. Take advantage of the command history
    Press the up key to recover the most recent command. Use up and down to scroll through the entire history.
  5. Take advantage of auto-completion
    When entering a command name or filepath, type just the first few characters and press tab to have the shell attempt to complete it. If ther are multiple possibilities, the shell will display them. Type a few more characters and press tab again, repeating as necessary. Once you get the hang of it, this makes dealing with long path names much easier.

Useful Text Programs

  1. Send text or files via email
    "mail" is the most basic text program for sending email, but it can't handle attachements, so for that you will need to use "mutt" instead.
    • Text body only, from a file: 
      mail [address] -s "subject" < text_file
    • Short text body only:
      echo "some text" | mail [address] -s "subject"
    • Short body plus attachment:
      echo "some text" | mutt [address] -s "subject" -a [attachment_file]

Shell Configuration

  1. Create aliases for command commands.
    Edit the ".aliases" file in your home directory. Follow the format used for the existing aliases.
  2. Enable file/folder coloring with "ls"
    Edit the ".aliases" file and add the following entry to the top (before any commands that use "ls"):
    • alias ls 'ls --color=auto'
    This makes navigation somewhat easier.
